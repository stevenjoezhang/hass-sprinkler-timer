# Sprinkler Timer for Home Assistant

一个轻量的 Home Assistant 自定义集成，用来创建多个喷淋定时器。每个定时器可以绑定多个 `switch` 实体，按固定时间和重复周期自动喷淋，并根据和风天气小时预报自动避开雨天。

## 功能

* 支持通过「设置 -> 设备与服务」创建任意多个喷淋定时器
* 每个定时器可绑定多个喷淋 `switch`
* 支持每天多个开始时间，例如 `09:00, 14:00`
* 支持设置每次喷淋时长和每几天重复一次
* 支持和风天气未来 24 小时预报
* 每小时保存未来 12 小时逐小时雨量，形成本地雨量时间线
* 喷淋前统计「过去 12 小时 + 未来 12 小时」累计雨量，超过阈值则跳过
* 首页只暴露少量实体：定时器开关、下次喷淋时间、最近决策

## 安装

### 通过 HACS 安装

仓库发布后，可以将本仓库作为 HACS 自定义仓库添加，类别选择 `Integration`。

### 手动安装

将本仓库中的 `custom_components/sprinkler_timer` 目录复制到 Home Assistant 配置目录：

```sh
cp -r hass-sprinkler-timer/custom_components/sprinkler_timer ~/hass/custom_components/
```

然后重启 Home Assistant。

## 配置

重启后进入：

```text
设置 -> 设备与服务 -> 添加集成 -> Sprinkler Timer
```

创建定时器时需要填写：

| 参数 | 说明 |
| - | - |
| 计划名称 | 定时器名称 |
| 绑定的喷淋 switch | 一个或多个用于控制喷淋设备的 `switch` 实体 |
| 和风天气 API Key | 用于调用 `/v7/weather/24h` 小时预报 |
| 和风天气 API Host | 默认 `https://devapi.qweather.com`；如您使用专属 API Host，可在这里填写 |
| 经纬度 | 默认读取 Home Assistant 的家庭位置，可手动调整 |
| 雨天自动跳过 | 是否启用雨量判断 |
| 24 小时窗口累计雨量阈值 | 过去 12 小时 + 未来 12 小时累计雨量达到该值时跳过 |
| 每天开始时间 | 多个时间用逗号分隔，例如 `09:00, 14:00` |
| 每次喷淋时长 | 单次喷淋持续分钟数 |
| 每几天重复一次 | 例如 `1` 表示每天，`2` 表示每两天 |
| 启用定时器 | 初始是否启用 |

## 暴露实体

每个定时器会暴露 3 个实体：

```text
switch.<plan>_定时器开关
sensor.<plan>_下次喷淋时间
sensor.<plan>_最近决策
```

`最近决策` 会记录执行或跳过原因，例如：

```text
已跳过：过去 12 小时到未来 12 小时累计降雨 4.2 mm，达到阈值 3 mm
已执行：2026-05-05 09:00 开始喷淋 10 分钟；无法获取降雨预报（请求超时），已按无雨继续喷淋
已执行：2026-05-05 09:00 开始喷淋 10 分钟
已完成：2026-05-05 09:10 停止喷淋
```

## 雨量算法

和风天气 API 只提供未来预报，因此本集成会维护一个本地雨量时间线：

```text
每小时请求一次未来 24 小时预报
取未来 12 小时每小时 precip
写入本地时间线
未来小时会随新预报滚动更新
过去小时保留为已保存历史
```

喷淋前会计算：

```text
过去 12 小时 + 未来 12 小时的累计雨量
```

如果累计雨量达到配置阈值，则跳过本次喷淋。

如果临近喷淋时无法获取降雨预报，集成会按“无雨”继续执行本次喷淋，并在 `最近决策` 中附带 `无法获取降雨预报` 提示。

## 图标

当前 Home Assistant 2026.3 之前的版本不会读取自定义集成目录内的本地图标，集成卡片图标通常来自 Home Assistant Brands。仓库中已包含：

```text
custom_components/sprinkler_timer/brand/icon.png
custom_components/sprinkler_timer/brand/logo.png
```

这些文件适用于支持本地品牌资源的新版本 Home Assistant。旧版本需要将品牌图片提交到 `home-assistant/brands` 才会在集成列表中显示。
